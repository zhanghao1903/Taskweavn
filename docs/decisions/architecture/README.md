# Architecture Decision Records

> Status: active directory
> Last Updated: 2026-05-18

Use Architecture Decision Records for core object boundaries, protocols, lifecycles, storage/replay models, trust model, and system communication boundaries.

Existing architecture decisions now live here. New architecture decisions should continue in this directory unless a later numbering scheme is introduced.

Examples:

- Authoring Domain vs Execution TaskBus.
- Session Message Stream with Task projection.
- UI/backend Query-Command-Event boundary.
- Workspace communication protocol over direct tool explosion.
