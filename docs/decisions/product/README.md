# Product Decision Records

> Status: active directory
> Last Updated: 2026-05-18

Use Product Decision Records for product principles, user experience commitments, explicit non-goals, and scope trade-offs that would be expensive to reverse.

Examples:

- Task-first UI over chat-first UI.
- Personal assistant scope instead of multi-user collaboration.
- Plato as user-facing name while TaskWeavn remains internal project name.
- Audit / Trust page as part of 1.0 P0.
