Tier 1 Design Philosophy Meta-principles for architecture decisions — align before coding ▼
ArchitectureSystem Design Principles
Action / Observation Symmetry
All operations are strongly-typed Actions; all feedback is symmetrical Observations. CodeAction is a subclass of Action and does not break the type system.
Runtime Decoupling
The agent core has no knowledge of where it is executing. Local process, Docker, and remote sandbox are all transparent to the agent; the interface never changes.
Single Agent First, Multi-Agent Reserved
The first deliverable is a complete single-agent. The multi-agent coordination layer (AutoGen-style) is reserved as an upper-layer abstraction and does not pollute the single-agent core.
Thought Side-Channel Storage
Thoughts are not first-class members of the EventStream; they are linked via event_id as a side channel. Write, replay, and retrieval capabilities exist and can be toggled via config.
ExecutionEngineering Principles
Configurable over Hardcoded
Thought writing, audit agent toggle, sandbox backend, storage backend — any capability that is not needed every time becomes a config option rather than a code branch.
Interface before Implementation
LLMClient, ThoughtStore, and AuditAgent define their Protocol first; concrete implementations come after. The multi-agent layer interface is defined at the end of Phase 1, even if the implementation is empty.
Observability Built-in
Structured logging, EventStream replay, and ThoughtStore queries are first-class capabilities from day one — not debugging tools bolted on later.
Audit Does Not Rely on LLM Self-discipline
CodeAction's tracking fields are enforced by the schema; the sandbox auto-instruments — no reliance on the LLM proactively writing logging code.

Phase 1 — Foundation (Week 1–3) · Goal: minimal runnable single agent
1.1Action / Observation Base Classes + Type RegistryP0Arch
Define BaseAction / BaseObservation dataclasses with common fields (event_id, timestamp, etc.) and an ActionRegistry that auto-collects subclasses. Also define OrchestratorProtocol stub for the multi-agent layer.
Output: types/Acceptance: all Actions serialise to JSON
1.2EventStream (in-memory)P0Arch
Implement append / iter / replay core methods with in-memory storage. Interface is identical to the future SQLite version. ThoughtStore Protocol defined simultaneously; NullThoughtStore as default.
Output: core/event_stream.pyAcceptance: full session replay works
1.3LLMClient Wrapper + Runtime InterfaceP0Exec
Wrap Anthropic SDK with streaming support and unified tool_use message format. Define Runtime Protocol (execute_action → Observation) and implement LocalRuntime.
Output: llm/ + runtime/Acceptance: model swap requires zero upper-layer changes
1.4Basic Tool Set (File + Shell)P0Exec
Implement ReadFileTool / WriteFileTool / ListDirTool / RunCommandTool; each produces its own Observation type. Path safety checks and command timeout control included.
Output: tools/fs/ + tools/shell/Acceptance: all have unit tests
1.5ReAct Main Loop + CLIP0Exec
Implement Thought→Action→Observation loop; events written to EventStream; max-steps limit and AgentFinishAction termination. Typer CLI with --model / --workspace flags.
Output: core/loop.py + cli/Acceptance: completes "write hello world and run it"
Phase 2 — CodeAction + Audit (Week 4–6) · Goal: safe, auditable code execution ✅ Complete
2.1CodeAction Schema DesignP1Arch
Define CodeAction (intent: str, code: str, language: str, tracking: TrackingConfig). TrackingConfig declares side-effect scope (file list, variable names). Corresponding CodeExecutionObservation structure.
Output: types/code_action.pyAcceptance: schema usable as LLM JSON Schema output
2.2Sandbox Executor + Auto-instrumentationP1Exec
subprocess sandbox; auto-injects file-diff snapshots and variable tracking logging before/after code based on TrackingConfig. No reliance on LLM writing logs. Timeout, memory limit, dangerous command interception.
Output: runtime/sandbox.pyAcceptance: all file changes fully recorded
2.3Audit Agent ImplementationP1Exec
LLM-driven AuditAgent. Input: intent + code + execution output. Output: structured facts (file changes, variable values) + verdict (matches intent / deviation description). Main agent consumes verdict and can trace back to raw facts.
Output: agents/audit_agent.pyAcceptance: false-positive rate < 10% (20 manual cases)
2.4ThoughtStore SQLite ImplementationP1Exec
Implement SqliteThoughtStore. ThoughtRecord: event_id / phase / content / timestamp / metadata. Supports replay by session_id and lookup by event_id. ThoughtConfig controls enabled / phases / backend.
Output: memory/thought_store.pyAcceptance: zero performance impact when disabled
Phase 3 — Memory + Persistence (Week 6–8) · Goal: full single-agent version
3.1ConversationHistory + Token BudgetP1Exec
Sliding-window truncation with tiktoken counting. Retains system prompt + last N turns + key Observations for the current task. Exceeding budget triggers priority-based pruning, not hard truncation.
Output: memory/history.py
3.2RAG Semantic Code RetrievalP1Exec
tree-sitter parses code structure; ChromaDB stores function/class embeddings. Agent auto-retrieves relevant code snippets before reasoning. Incremental indexing; auto-updates on file modification.
Output: memory/rag.py
3.3EventStream Persistence + Session ResumeP1Exec
SQLite EventStream with an interface identical to the in-memory version (Protocol guarantee). Supports resuming sessions from an interruption point; past tasks can be fully replayed.
Output: core/event_stream_sqlite.pyAcceptance: in-memory / SQLite hot-switchable
3.4Benchmark Evaluation + Test SuiteP1Arch
Design 10–20 standard coding tasks; auto-run and record success rate / token usage / step count. ThoughtStore accumulates evaluation data automatically. Thought + Event jointly recorded for later analysis.
Output: scripts/benchmark.pyMilestone: full single-agent version delivered

4.1AgentOrchestrator ImplementationP2Arch
